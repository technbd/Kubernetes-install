## Multi-Node HA Cluster:

Here’s a real production-style 3-node HA k0s cluster setup using the recommended tool `k0sctl` (this is the cleanest and most reliable way).





### Architecture Overview:

```
                ┌─────────────────────┐
                │   k0sctl (Admin Pc) │
                └─────────┬───────────┘
                          │ SSH
        ┌─────────────────┼─────────────────┐
        │                 │                 │
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│ node-1:      │ │ node-2:      │ │ node-3:      │
│ controller   │ │ controller   │ │ controller   │
│ + worker     │ │ + worker     │ │ + worker     │
│ (k0s + etcd) │ │ (k0s + etcd) │ │ (k0s + etcd) │
└──────┬───────┘ └──────┬───────┘ └──────┬───────┘
       │                │                │
       └─────── HAProxy Layer ─────────┘
                   │
        ┌──────────────────────┐
        │ VIP 192.168.10.194   │
        │ Keepalived (VRRP)    │
        └──────────────────────┘
```


### IP Allocation: 

| Role | Hostname | IP |
| ---- | -------- | --- |
| Controller + Worker | worker1 | 192.168.10.191 |
| Controller + Worker | worker2 | 192.168.10.192 |
| Controller + Worker | worker3 | 192.168.10.193 | 
| VIP (keepalived + HAProxy) | lb01 | 192.168.10.194 | 


### Prerequisites: 

- 3 Linux VMs (e.g., Rocky Linux, Ubuntu) that can communicate with each other.
- Static IP addresses assigned to each node.
- A Load Balancer (like **HAProxy** or an **AWS ELB**) sitting in front of the controllers to balance traffic on port `6443` (Kubernetes API) and port `8132` (k0s request routing).

## Install K0s Cluster:

`k0sctl` is a command-line tool for bootstrapping and managing `k0s` clusters. k0sctl connects to the provided hosts using SSH and gathers information on the hosts, with which it forms a cluster by configuring the hosts, deploying k0s, and then connecting the k0s nodes together.


## Method-1:  `k0sctl` (Recommended)

### Install `k0sctl` (on admin machine): 

`k0sctl` automates the entire HA setup from your local/admin machine via SSH.


_Run on local/admin machine:_
```
curl -sSLf https://github.com/k0sproject/k0sctl/releases/latest/download/k0sctl-linux-amd64 -o /usr/local/bin/k0sctl

chmod +x /usr/local/bin/k0sctl
```


```
k0sctl version

version: v0.30.1
commit: 4afdb2b
```


### Create k0s cluster config:

```yaml
## k0s-cluster.yaml

apiVersion: k0sctl.k0sproject.io/v1beta1
kind: Cluster
metadata:
  name: ha-cluster
spec:
  hosts:
    - ssh:
        address: 192.168.10.191
        user: root
        port: 22
      role: controller+worker
      noTaints: true              # ← allow workloads on this node
      privateAddress: 192.168.10.191    # ← force this IP, ignore vip 192.168.10.194

    - ssh:
        address: 192.168.10.192
        user: root
        port: 22
      role: controller+worker
      noTaints: true
      privateAddress: 192.168.10.192

    - ssh:
        address: 192.168.10.193
        user: root
        port: 22
      role: controller+worker
      noTaints: true
      privateAddress: 192.168.10.193

  k0s:
    version: v1.35.4+k0s.0
    config:
      apiVersion: k0s.k0sproject.io/v1beta1
      kind: ClusterConfig
      metadata:
        name: ha-cluster
      spec:
        api:
          #address: 192.168.10.191  # first controller IP
          externalAddress: 192.168.10.194   # use VIP (192.168.10.194) keepalived is already running! Use it
          sans:
            - 192.168.10.194
            - 192.168.10.191
            - 192.168.10.192
            - 192.168.10.193
        network:
          provider: calico
          podCIDR: 10.244.0.0/16
          serviceCIDR: 10.96.0.0/12
        storage:
          type: etcd                         # embedded etcd for HA

```

On the Controller-1 machine, generate a secure token that allows worker nodes to connect to it:

```

```

scp /tmp/controller1-token.txt root@192.168.10.193:/tmp/controller1-token.txt

### Remove Taints (allow workloads):

Remove taint from all controller+worker nodes. The `-` at the end means remove the taint.


```
kubectl describe nodes | grep -A5 Taints


Taints:             node-role.kubernetes.io/control-plane:NoSchedule
Unschedulable:      false
Lease:
  HolderIdentity:  worker1
  AcquireTime:     <unset>
  RenewTime:       Mon, 15 Jun 2026 14:02:04 +0600
--
Taints:             node-role.kubernetes.io/control-plane:NoSchedule
Unschedulable:      false
Lease:
  HolderIdentity:  worker2
  AcquireTime:     <unset>
  RenewTime:       Mon, 15 Jun 2026 14:02:04 +0600
--
Taints:             node-role.kubernetes.io/control-plane:NoSchedule
Unschedulable:      false
Lease:
  HolderIdentity:  worker3
  AcquireTime:     <unset>
  RenewTime:       Mon, 15 Jun 2026 14:02:01 +0600
```



_Remove from All Nodes at Once:_
```
k0s kubectl taint nodes --all node-role.kubernetes.io/control-plane:NoSchedule-
```



_Or, Remove Taints from All 3 Nodes:_
```
k0s kubectl taint nodes worker1 node-role.kubernetes.io/control-plane:NoSchedule-
k0s kubectl taint nodes worker2 node-role.kubernetes.io/control-plane:NoSchedule-
k0s kubectl taint nodes worker3 node-role.kubernetes.io/control-plane:NoSchedule-
```


_Verify Current Taints:_
```
kubectl describe nodes | grep Taints
kubectl describe nodes | grep -A5 Taints
```

## Deploy a Test Workload:

```
# Deploy nginx:
kubectl create deployment nginx --image=nginx --replicas=3
kubectl expose deployment nginx --port=80 --type=NodePort

# Stream events in real time:
kubectl get events -w

# Restart a Deployment:
kubectl rollout restart deployment nginx
kubectl rollout status deployment nginx


# Check pods spread across all 3 nodes:
kubectl get pods -o wide

# Find the Assigned Port:
kubectl get service nginx

# Test it out:
curl http://192.168.10.194:30269


# Cleanup:
kubectl delete deployment nginx
kubectl delete service nginx
```
