## Multi-Node cluster:

Here’s the standard production-style k0s cluster setup: 1 master (controller) + 2 worker nodes using k0sctl.

This is the most common real-world topology.


### 


### Architecture Overview:

```
            ┌──────────────────────┐
            │   k0sctl (admin PC)  │
            └─────────┬────────────┘
                      │ SSH
        ┌─────────────┴─────────────┐
        │                           │
┌──────────────┐          ┌──────────────┐
│  Master      │          │ Worker-1     │
│ (Control)    │          │ Node         │
│ API + etcd   │          └──────────────┘
└──────────────┘          ┌──────────────┐
                          │ Worker-2     │
                          │ Node         │
                          └──────────────┘
```


### Assumptions: 

| Role | Hostname | IP |
| ---- | -------- | --- |
| Controller | master1 | 192.168.10.191 |
| Worker 1 | worker1 | 192.168.10.192 |
| Worker 2 | worker2 | 192.168.10.193 | 





### Prepare All Nodes:

_Set Hostname:_
```
hostnamectl set-hostname master1
hostnamectl set-hostname worker1
hostnamectl set-hostname worker2
```


_Enable Cgroup v2:_
```
grubby --update-kernel=ALL --args="systemd.unified_cgroup_hierarchy=1"

reboot
```


```
stat -fc %T /sys/fs/cgroup/
```



_Disable swap:_
```
swapoff -a
```



_Enable time sync:_
```
dnf install -y chrony
```


```
systemctl restart chronyd
systemctl enable chronyd
systemctl status chronyd
```

### Install k0s (ALL Machines):

```
curl --proto '=https' --tlsv1.2 -sSf https://get.k0s.sh | sudo sh
```


```
k0s version
```




### Configure & Start Controller (on Master only): 

On your Controller machine, install and start it strictly as a control plane (no worker workloads):


_Create a configuration file:_
```
mkdir -p /etc/k0s

# Generate default config: 
k0s config create > /etc/k0s/k0s.yaml
```


```
k0s install controller

k0s install controller -c /etc/k0s/k0s.yaml


## Or, Multi-node where controller also runs pods:
k0s install controller --enable-worker
```


```
k0s start
```


_Verify:_
```
k0s status

Version: v1.35.4+k0s.0
Process ID: 24311
Role: controller
Workloads: false
SingleNode: false
```


```
systemctl status k0scontroller
```



### Generate a join token:

On the Controller machine, generate a secure token that allows worker nodes to connect to it:

```
k0s token create --role=worker

k0s token create --role=worker > /tmp/worker-token.token
```


_Transfer tokens to worker nodes:_
```
scp /tmp/worker-token.token root@192.168.10.192:/tmp/worker.token
scp /tmp/worker-token.token root@192.168.10.193:/tmp/worker.token
```



### Join Worker Nodes: 

Switch over to your Worker machine. Run the install command while passing the token you just copied:

```
k0s install worker --token-file /tmp/worker.token

k0s start
```


```
k0s status


Version: v1.35.4+k0s.0
Process ID: 4221
Role: worker
Workloads: true
SingleNode: false
Kube-api probing successful: true
Kube-api probing last error:
```


```
systemctl status k0sworker
```

### Get kubeconfig (optional):

_Export the kubeconfig (on Master):_
```
k0s kubeconfig admin > ~/.kube/config
chmod 600 ~/.kube/config
```


_To use from your local/admin machine:_
```
mkdir -p ~/.kube

scp root@<admin_ip>:~/.kube/config ~/.kube/config

export KUBECONFIG=.kube/config
```
