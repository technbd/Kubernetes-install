## k0rdent:

`k0rdent` is an open-source **platform engineering / Kubernetes management system** developed by **Mirantis**. It’s designed to help organizations manage many Kubernetes clusters across cloud, on-prem, and edge environments from a single control plane.


_k0rdent solves this by providing:_
- Central control
- Standard templates
- Automation for cluster operations
- Multi-cloud consistency


### What it does?

_k0rdent focuses on multi-cluster Kubernetes lifecycle management, meaning it can:_
- Create Kubernetes clusters (on AWS, Azure, OpenStack, bare metal, etc.)
- Upgrade and maintain clusters automatically
- Delete or scale clusters when needed
- Apply policies and standard configurations across all clusters
- Provide a consistent “golden path” for developers


### Main components:

_k0rdent is usually described as three big parts:_
- **KCM (k0rdent Cluster Manager)**: Handles creating and managing Kubernetes clusters (lifecycle: create, update, delete)
- **KSM (k0rdent State Manager)**: Manages workloads and services running on clusters (like Helm apps, policies)
- **KOF (k0rdent Observability & FinOps)**: Handles monitoring, logs, metrics, and cost tracking across clusters



### Architecture view (simplified): 

```
            k0rdent Management Cluster
          (running on k0s or other K8s)

        ┌──────────────────────────────┐
        │  k0rdent controllers (KCM)   │
        │  k0rdent state (KSM)         │
        │  observability (KOF)         │
        └───────────┬──────────────────┘
                    │
     ┌──────────────┼──────────────┐
     │              │              │
 k0s cluster    k0s cluster   k0s cluster
 (dev)          (staging)     (prod)
```



### Prerequisites: 
For a production installation:
- Management Cluster: 
    - Kubernetes v1.30+
    - k0s v1.34+ or newer
    - kubectl access
    - Helm 3.x
- A minimum of 8GB RAM, 4 vCPUs, 100GB SSD (for example, AWS t2.xlarge or equivalent)

## Install k0rdent: 


### Install k0s:

Run the following commands to install and start k0s as a single-node controller + worker:

```
curl --proto '=https' --tlsv1.2 -sSf https://get.k0s.sh | sudo sh

k0s version

k0s install controller --enable-worker --no-taints

k0s start
k0s status

k0s kubectl get nodes
```


```
k0s kubeconfig admin > ~/.kube/config

export KUBECONFIG=~/.kube/config
```


### Install kubectl:

k0s installs a compatible `kubectl` and makes it accessible via its own client.

_Add the Kubernetes Repository:_
```
cat <<EOF | sudo tee /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=https://pkgs.k8s.io/core:/stable:/v1.35/rpm/
enabled=1
gpgcheck=1
gpgkey=https://pkgs.k8s.io/core:/stable:/v1.35/rpm/repodata/repomd.xml.key
EOF
```


```
dnf install -y kubectl

kubectl version --client
```


```
kubectl get nodes
```


```
kubectl get events -A -w
```



### Install Helm: 

The easiest way to install k0rdent is through its Helm chart, so install Helm first:

```
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

helm version
```



### Install k0rdent via Helm: 

Install `k0rdent` using its Helm chart (use Helm v3) and this deploys `k0rdent` into the `kcm-system` namespace on your `k0s` cluster: 

```
kubectl create namespace kcm-system
kubectl get ns
```


```
helm install kcm oci://ghcr.io/k0rdent/kcm/charts/kcm --version 1.10.0 -n kcm-system --timeout 10m
helm install kcm oci://ghcr.io/k0rdent/kcm/charts/kcm --version 1.10.0 -n kcm-system --create-namespace --timeout 10m
```

_If uninstall, Clean up and retry:_
```
helm list -A

kubectl delete management.k0rdent kcm
helm uninstall kcm -n kcm-system
kubectl delete ns kcm-system
```



_Verify KCM:_
```
kubectl get pods -n kcm-system
```


_Verify ProjectSveltos (KSM):_
```
kubectl get pods -n projectsveltos
```
 

_Verify K0rdent Ready:_
```
kubectl get Management -n kcm-system
```


_Verify provider and cluster templates:_
```
kubectl get providertemplate -n kcm-system

kubectl get clustertemplate -n kcm-system
```








`k0rdent` is a Kubernetes-native platform that acts like a centralized “control plane of control planes” for managing clusters at scale.
