## Kubernetes Monitoring using Prometheus & Grafana: 

Kubernetes monitoring is the process of collecting, storing, and visualizing metrics from a Kubernetes cluster to ensure performance, reliability, and health visibility.


_A typical monitoring stack uses:_
- Prometheus → collects and stores metrics
- Grafana → visualizes metrics in dashboards
- kube-state-metrics → exposes Kubernetes object status
- Node Exporter → exposes node-level system metrics
- Alertmanager → sends alerts (Slack, email, etc.)



### Monitoring Architecture: 

```
Kubernetes Cluster
   |
   |--> Node Exporter (Node metrics)
   |--> kube-state-metrics (K8s objects)
   |--> Application metrics
   |
Prometheus (data collection + storage)
   |
Grafana (dashboards + visualization)
   |
Alertmanager (alerts/notifications)
```

## Local-path Provisioner: 

When you install Prometheus (or Grafana) via Helm, those charts request **PersistentVolumeClaims (PVCs)** — storage that survives pod restarts. Kubernetes doesn't provision that storage automatically; it needs a **StorageClass** backed by a provisioner to actually create the disk.

On managed cloud clusters (EKS, GKE, AKE), a default StorageClass already exists (EBS, GCE PD, etc.), so PVCs just work. But on:

- kubeadm clusters
- k3s / k3d
- kind
- bare-metal / VMs

…there's no default StorageClass, so PVC requests stay in Pending forever, and the pod never starts: `pod has unbound immediate PersistentVolumeClaims`


It watches for PVC requests and dynamically creates a directory on the node's local filesystem (e.g. `/opt/local-path-provisioner/`) and binds a PV to it. It also registers itself as the default StorageClass, so any chart that requests storage gets it automatically.


The StorageClass definition in the manifest has no `is-default-class` annotation, so it installs as a **non-default StorageClass**. The patch adds that annotation — which is correct in intent.

```
kubectl apply -f https://raw.githubusercontent.com/rancher/local-path-provisioner/master/deploy/local-path-storage.yaml
```


No default is set — the (`default`) tag is absent from the NAME column. So patch is needed: 

```
kubectl get storageclass

NAME         PROVISIONER             RECLAIMPOLICY   VOLUMEBINDINGMODE      ALLOWVOLUMEEXPANSION   AGE
local-path   rancher.io/local-path   Delete          WaitForFirstConsumer   false                  56s
```


_Only then patch if no default exists:_
```
kubectl patch storageclass local-path -p '{"metadata": {"annotations":{"storageclass.kubernetes.io/is-default-class":"true"}}}'
```


```
kubectl get storageclass


NAME                   PROVISIONER             RECLAIMPOLICY   VOLUMEBINDINGMODE      ALLOWVOLUMEEXPANSION   AGE
local-path (default)   rancher.io/local-path   Delete          WaitForFirstConsumer   false                  2m29s
```



```
kubectl get pvc
kubectl get pv
```

## Install Prometheus: 


### Add Prometheus Helm repository:

```
helm repo add <DESIRED-NAME> <HELM-REPO-URL>
```


```
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
```


```
helm repo list

helm repo update

helm search repo prometheus-community/prometheus
```



### Install Prometheus via Helm: 


```
helm install my-prometheus prometheus-community/prometheus 
```


```
kubectl get deploy
kubectl get pods
kubectl get daemonsets
kubectl get svc
```



### If PersistentVolumeClaims issues: 

If error like: `pod has unbound immediate PersistentVolumeClaims` disable persistence completely (Lab setup): 



```
helm show values prometheus-community/prometheus > values.yaml
```


_If you don’t need persistence edit Helm values:_
```yaml
## prometheus chart:

server:
  persistentVolume:
    enabled: false

## Also check:
alertmanager:
  persistence:
    enabled: false
```



```
helm install my-prometheus prometheus-community/prometheus -f values.yaml
```




### Expose Prometheus:


_NodePort → Service Port → Container Port:_
- `--port`	: Service port (what Service exposes internally)
- `--target-port` :	Container port inside Pod


_Syntax:_
```
kubectl expose deployment my-prometheus-server --type=NodePort --port=<service-port> --target-port=<pod-container-port>
```


_Expose prometheus service:_
```
kubectl expose deployment my-prometheus-server --type=NodePort --port=80 --target-port=9090 --name=prometheus-server-svc
```


Or, 
```
kubectl port-forward --address 0.0.0.0 svc/my-prometheus-server 9999:80
```

## Install Grafana: 


### Add Grafana Helm repository:


```
helm repo add grafana-community https://grafana-community.github.io/helm-charts
```


```
helm repo list
helm search repo grafana-community/grafana
```



### Install Grafana via Helm:


```
helm install my-grafana grafana-community/grafana
```

```
kubectl get secret --namespace default my-grafana -o jsonpath="{.data.admin-password}" | base64 --decode ; echo

```

## Install Prometheus-stack (Production recommended): 

**kube-prometheus-stack** is a complete Kubernetes monitoring solution built on the Prometheus Operator. Instead of installing Prometheus, Grafana, Alertmanager, and exporters separately, it deploys and configures them as a unified stack.

Compared to the standalone Prometheus chart, kube-prometheus-stack provides production-ready monitoring with preconfigured dashboards, alerts, ServiceMonitors, and PodMonitors.

It includes:
- Prometheus for metrics collection
- Grafana for dashboards and visualization
- Alertmanager for alerting
- Node Exporter for node-level metrics
- kube-state-metrics for Kubernetes object metrics
- Prometheus Operator for automated Prometheus management



### Install Prometheus-stack via Helm: 


```
helm repo list
helm search repo prometheus-community/kube-prometheus-stack
```


```
helm install my-prometheus prometheus-community/kube-prometheus-stack --set alertmanager.enabled=false --timeout 15m
```


_Get admin password:_
```
kubectl --namespace default get secrets my-prometheus-grafana -o jsonpath="{.data.admin-password}" | base64 -d ; echo
```



_If Storage Persistent issues:_
```
helm show values prometheus-community/kube-prometheus-stack > values.yaml
```


_For kube-prometheus-stack:_
```yaml
prometheus:
  prometheusSpec:
    storageSpec: {}
```



```
helm install my-prometheus prometheus-community/kube-prometheus-stack -f values.yaml
```



### Expose Port: 

```
kubectl get svc
```


```
kubectl port-forward --address 0.0.0.0 svc/prometheus-operated 9999:9090
```


```
kubectl port-forward --address 0.0.0.0 svc/my-prometheus-grafana 3333:80
```





### Charts Uninstall: 

```
helm uninstall my-prometheus
helm uninstall my-grafana
```
