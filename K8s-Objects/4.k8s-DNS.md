## Kubernetes DNS:

Kubernetes DNS is the **built-in service discovery system** that allows Pods and Services to communicate using names instead of IP addresses. It is powered by **CoreDNS**. 




### Why Kubernetes DNS is Important:

In Kubernetes, Pods are ephemeral (they can be created, deleted, or moved anytime). Their IP addresses change frequently.

## DNS Name Formats:

Kubernetes automatically manages DNS records for Services and Pods based on a strict naming convention.


Kubernetes can create DNS records for Pods, but the behavior depends on whether you're using:

- A regular Pod
- A Deployment
- A StatefulSet
- A Headless Service


### 1. DNS Records for Services: 

### A. Normal (ClusterIP) Service:

Normal Services (with a ClusterIP) get a DNS A record mapping to their internal IP: `<service-name>.<namespace>.svc.cluster.local`


```
kubectl create ns prod
```


```
kubectl get ns prod

NAME   STATUS   AGE
prod   Active   12s
```



_Create a Deployment YAML File:_
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  namespace: prod
  labels:
    app: web-server

spec:
  replicas: 3          # Tells Kubernetes to run 3 identical copies of this Pod
  selector:
    matchLabels:
      app: web-server  # Tells the deployment which Pods to manage

  template:
    metadata:
      labels:
        app: web-server # This label must match the selector above

    spec:
      containers:
      - name: nginx-container
        image: nginx:1.25
        ports:
        - containerPort: 80 
        resources: {} 
```


_Create a Service YAML File:_
```yaml 
apiVersion: v1
kind: Service
metadata:
  name: nginx-deployment-svc
  namespace: prod

spec:
  type: ClusterIP      # Defines the service type NodePort or LoadBalancer
  selector:
    app: web-server    # Must match the template labels in the Deployment

  ports:
  - protocol: TCP
    port: 80           # Port the Service exposes
    targetPort: 80     # Port the containers are listening on
```

kubectl apply -f nginx-deployment.yaml

kubectl apply -f nginx-deployment-svc.yaml

### B. Headless Service (no ClusterIP): 

If a Service is configured with `clusterIP: None` (Headless), CoreDNS does not return a single ClusterIP. Instead, it returns an **A record** containing the list of IPs of all the backend Pods matching the service selector.

- Normal Service : Returns 1 Cluster IP
- Headless Service : Returns Pod IPs directly


_A headless service is used for:_
- StatefulSet (MySQL, Redis cluster, Kafka)
- Direct Pod discovery
- Peer-to-peer communication

_NOT for:_
- simple HTTP load balancing
- frontend → backend routing


#### Deployment vs StatefulSet: 

If you are using a Deployment, the `serviceName` field does not exist and should not be used. A StatefulSet requires `serviceName` because each Pod has a unique identity and stable DNS name.

| Resource                     | `serviceName` Required?             |
| ---------------------------- | ----------------------------------- |
| Deployment                   | ❌ No                                |
| StatefulSet                  | ✅ Yes                               |
| StatefulSet + stable Pod DNS | ✅ Must reference a Headless Service |
| Headless Service             | Recommended for StatefulSets        |



_Create a StatefulSet YAML File:_
```yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: nginx-statefulset
  namespace: prod

spec:
  serviceName: nginx-headless-svc  # Must match the Headless Service name ('serviceName' field required for StatefulSet)
  replicas: 3
  selector:
    matchLabels:
      app: nginx-web  

  template:
    metadata:
      labels:
        app: nginx-web 

    spec:
      containers:
      - name: nginx-container 
        image: nginx:1.25
        ports:
        - containerPort: 80  # Default Nginx port
```




_Create a Service YAML File:_
```yaml 
apiVersion: v1
kind: Service
metadata:
  name: nginx-headless-svc   # Must match the 'serviceName' in the StatefulSet
  namespace: prod

spec:
  clusterIP: None       # Defines this as a Headless Service
  selector:
    app: nginx-web      # Must exactly match the template labels

  ports:
    - port: 80
      targetPort: 80
```



```
kubectl apply -f nginx-statefulset.yaml

kubectl apply -f nginx-headless-svc.yaml
```

### 2. DNS Records for Pods:

By default, DNS records for pods are not created. Pod IPs change when Pods are recreated. Old DNS record becomes invalid.

By default, individual Pods are assigned a DNS record structured by their IP address:  `<pod-ip-address>.<namespace>.pod.cluster.local`


_Kubernetes may create a Pod DNS record like:_
```
10-244-0-110.prod.pod.cluster.local
```


_Verify:_
```
nslookup 10-244-0-110.prod.pod.cluster.local
```
