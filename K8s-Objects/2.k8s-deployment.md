## Kubernetes Deployment: 

A Deployment is a Kubernetes object that **manages a stateless application** running on Pods. It ensures that a specified number of Pod replicas are running and provides features like rolling updates, rollbacks, scaling, and self-healing.

Under the hood, a Deployment doesn't manage Pods directly. It manages a ReplicaSet, and the ReplicaSet manages the Pods.


_What a Deployment gives you:_
- Self-healing — if a Pod dies or a node fails, it's replaced automatically
- Declarative updates — change the spec, Kubernetes handles the transition
- Rollback — revert to any previous revision
- Scaling — change replica count on demand
- Pausing/resuming rollouts for controlled releases


### Why Use a Deployment?

Without a Deployment, you would need to manually create and manage Pods. A Deployment automatically:

- Maintains the desired number of replicas
- Recreates failed Pods
- Performs rolling updates with zero or minimal downtime
- Supports rollback to previous versions
- Scales applications up or down easily
- Zero-Downtime Updates



### Deployment Architecture:

```
Deployment
    │
    ▼
ReplicaSet
    │
    ▼
Pods
```

## Deployment Manifest:


### Imperative Method:

Imperative commands allow you to create and manage Kubernetes resources directly from the command line without writing YAML files.


_Syntax:_
```
kubectl create deployment <deployment_name> --image=<image_name> --replicas=<number_of_pod>
```


_Create a deployment directly:_
```
kubectl create deployment nginx-deployment --image=nginx:1.25

kubectl create deployment nginx-deployment --image=nginx:1.25 --replicas=3

kubectl create deployment webapp-deployment --image=kodekloud/webapp-color:v1 --port=8080
```



_Generate YAML from Imperative Command:_
```
kubectl create deployment nginx-deployment --image=nginx:1.25 --replicas=3 --port=80 --dry-run=client -o yaml
kubectl create deployment nginx-deployment --image=nginx:1.25 --replicas=3 --port=80 --dry-run=client -o yaml > nginx-deployment.yaml
```


_Apply Configuration:_
```
kubectl apply -f nginx-deployment.yaml
```




### Declarative Method:

_Create a YAML File:_
```yaml 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx-web-server

spec:
  replicas: 3          # Tells Kubernetes to run 3 identical copies of this Pod
  selector:
    matchLabels:
      app: web-server  # Tells the Deployment which Pods belong to it

  template:
    metadata:
      labels:
        app: web-server # This label must match the selector above

    spec:
      containers:
      - name: nginx-container
        image: nginx:1.25
        ports:
        - containerPort: 80 
        resources: {} 
```


_Apply Configuration:_
```
kubectl apply -f nginx-deployment.yaml
```

### Deployment Strategy:


**RollingUpdate (Default):**
- Creates new Pods gradually.
- Removes old Pods gradually.
- Minimizes downtime.

```
strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 25%
      maxUnavailable: 25%
```


**Recreate:**
- Deletes all old Pods first.
- Creates new Pods afterward.
- Causes downtime.

```
strategy:
  type: Recreate
```

### Scaling a Deployment:

By default, Kubernetes creates 1 replica.


_Scale to 5 Replicas or update YAML (`replicas: 5`):_
```
kubectl scale deployment nginx-deployment --replicas=5 

Or,
kubectl scale deployment/nginx-deployment --replicas=3
```


_Autoscaling with HPA (Horizontal Pod Autoscaler) — scales replicas based on observed CPU/memory or custom metrics:_
- HPA will NOT work properly unless CPU requests are set in Deployment.

```
kubectl autoscale deployment/nginx-deployment --min=3 --max=5 --cpu-percent=80
kubectl autoscale deployment/nginx-deployment --min=3 --max=5 --cpu-percent=80 --dry-run=client -o yaml

Or,
kubectl autoscale deployment/nginx-deployment --min=3 --max=5 --cpu=80
kubectl autoscale deployment/nginx-deployment --min=3 --max=5 --cpu=80 --dry-run=client -o yaml
```





### Rolling Updates:

_Find the container name:_
```
kubectl get deployment nginx-deployment -o yaml | egrep "container|name"
```


_If update the image version:_
```
kubectl set image deployment/nginx-deployment nginx-container=nginx:1.26
```


_Add Change Cause:_
```
kubectl annotate deployment nginx-deployment kubernetes.io/change-cause="Updated nginx-container image to nginx:1.26" --overwrite
```



_Monitor rollout:_
```
kubectl rollout status deployment/nginx-deployment
```


_View rollout history:_
```
kubectl rollout history deployment/nginx-deployment

kubectl rollout history deployment/nginx-deployment --revision=1

kubectl rollout history deployment/nginx-deployment --revision=2
```



### Rollback: 

_Rollback to the previous version:_
```
kubectl rollout undo deployment/nginx-deployment
```


_Rollback to a specific revision:_
```
kubectl rollout undo deployment/nginx-deployment --to-revision=2
```

### Useful Commands:

```
-- Check Deployments:
kubectl get deployments


-- Check Pods: 
kubectl get pods


-- Check ReplicaSets: 
kubectl get rs


-- Get Deployment YAML: 
kubectl get deployment <deployment_name> -o yaml


-- Describe Deployment: 
kubectl describe deployment <deployment_name>


-- Edit Deployment: 
kubectl edit deployment <deployment_name>


-- Restart Deployment:
kubectl rollout restart deployment <deployment_name>


-- Watch Rollout: 
kubectl rollout status deployment <deployment_name>


-- Expose Pod/Deployment as a Service: 
kubectl expose pod <pod_name> --type=ClusterIP | NodePort | LoadBalancer --port=<service-port> --target-port=<container-port> --name=<svc-name-pod>
kubectl expose deployment <deployment_name> --type=ClusterIP | NodePort | LoadBalancer --port=<service-port> --target-port=<container-port> --name=<svc-name-deploy>


-- Port Forwarding: 
kubectl port-forward --address 0.0.0.0 deployment/nginx-deployment 80:80              [-> For deployment]
kubectl port-forward --address 0.0.0.0 pod/nginx-deployment-677b756b9d-8kgj9 80:80    [-> For single Pod]
kubectl port-forward --address 0.0.0.0 svc/nginx-svc 80:80                            [-> For SVC]


-- Delete Deployment: 
kubectl delete deployment <deployment_name>
```
